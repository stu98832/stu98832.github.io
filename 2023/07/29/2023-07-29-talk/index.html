久違的新文章

發表於 2023-07-29 分類於 雜談

距離上次寫的文章已經是大約 4 個月前的事情了
因為工作的關係，加上最近沒什麼動力
於是原本搞得像月更的 Design Pattern 又中斷了
寫這篇只是做個紀錄表示自己還活著 (x

一點雜談

現在都在寫 Typescript
用著不怎麼好用但又沒有其他替代品的 typeorm
即使是寫後端也大多是撰寫常見的 CRUD
時間都花在檔案該擺哪裡、如何分類、DB Table 設計等等的小問題
遇到邏輯的部份卻因為自己設計的 DB 結構而受限
最後測試難以撰寫，導致程式碼品質下降

這陣子我感覺不出有多大的提昇
目前還沒有的目標
現在生活感覺到的就是無趣、乏味
處於完全不知要幹麻的狀態

有關 Design Pattern 系列

這個系列估計是我寫最久而且最常虎頭蛇尾的系列了
起初是想用來紀錄學習的過程
但經過了前輩的題點，所以想說以實做為主
然而後來才發現，實踐比想像中還要困難

對於非常迷你的專案，幾乎沒有使用 Design Pattern 的必要
對於小型專案，實際上能使用的模式也是非常有限的
於是有很長一段時間沒有更新，之後就來到了今年一月開始的系列重寫

然而跟之前相比，不過是多寫了 template-method pattern 和 strategy pattern
本來下一篇要寫 factory pattern，但之後工作跟回家打打 Game 時間就花掉了 :)
總之就是，沒那個動力跟具體的目標去更新文章，然後又荒廢了

目前還沒有其他內容可以撰寫
我感覺要在工作的專案上應用模式是相當困難的事情
這非常吃經驗以及專案規模
對於一個幾乎都在寫 CRUD 的後端來說
設計模式除了拿來設計框架以外，基本上沒什麼太大的用處
至少目前專案的走向還看不出有什麼使用的必要性

最後這個系列應該也是無限期停更了

其他

其實只更新 Design Pattern 也是蠻怪的
最初建立這個筆記的目的其實只是想說用來面試加分用的
所以或想要生一點有品質的文章 (現在覺得含金量其實蠻低的…)
不過因為我自己本身沒什麼具體的目標驅使我寫 Code，或找份專案
因此我就少碰到很多問題，這導致了我自己沒辦法產出什麼文章，見識太少了

現況

目前工作下來感覺到的是疲累以及乏味的感覺
只是寫著有 CRUD 功能的 API ，幾乎沒什麼核心邏輯
但因為一些原因，導致大多都是 CRUD 功能的 Backend，卻要花費很長一段時間來實做

工作之後確實有了解一些東西
一些 CI/CD、K8S、Unit test、GraphQL 等等相關知識都有顯著提昇，不過都只是入門而已
目前沒有一個目標跟計畫去學更深入的內容
除了 Unit Test 算是一種基本能力以外，完全不知道能拿這些東西來做什麼

現在沒什麼動力去做額外的事情了
本來一段時間會更新的楓谷私服現在也處於停擺狀態
變成回家只會耍廢玩遊戲看影片的宅宅而已
倒也不是說對程式沒熱情，只是沒有一個目標驅使自己前進而已…

這篇文章之後可能又是長時間的停更了
完全不知道要寫什麼…
留在腦袋瓜裡的，就只有自己還沒辦法處理的問題而已…
